Class Hierarchy

  • SocketServer.BaseRequestHandler: Base class for request handler classes.
    • modbus_tk.simulator.RpcHandler: An instance of this class is created every time an RPC call is received by the server
  • modbus_tk.modbus.Databank: A databank is a shared place containing the data of all slaves
  • logging.Filterer: A base class for loggers and handlers which allows them to share common code.
    • logging.Handler: Handler instances dispatch logging events to specific destinations.
      • modbus_tk.utils.ConsoleHandler: This class is a logger handler.
      • modbus_tk.utils.DummyHandler: This class is a logger handler.
      • modbus_tk.utils.LogitHandler: This class is a logger handler.
  • modbus_tk.modbus.Master: This class implements the Modbus Application protocol for a master To be subclassed with a class implementing the MAC layer
    • modbus_tk.modbus_rtu.RtuMaster: Subclass of Master.
    • modbus_tk.modbus_tcp.TcpMaster: Subclass of Master.
  • modbus_tk.modbus.ModbusBlock: This class represents the values for a range of addresses
  • modbus_tk.modbus.Query: Interface to be implemented in subclass for every specific modbus MAC layer
    • modbus_tk.modbus_rtu.RtuQuery: Subclass of a Query.
    • modbus_tk.modbus_tcp.TcpQuery: Subclass of a Query.
  • modbus_tk.modbus.Server: This class owns several slaves and defines an interface to be implemented for a TCP or RTU server
    • modbus_tk.simulator.CompositeServer: make possible to have several servers sharing the same databank
    • modbus_tk.modbus_rtu.RtuServer: This class implements a simple and mono-threaded modbus rtu server
    • modbus_tk.modbus_tcp.TcpServer: This class implements a simple and mono-threaded modbus tcp server
  • modbus_tk.simulator.Simulator: The main class of the app in charge of running everything
  • modbus_tk.simulator_rpc_client.SimulatorRpcClient: Make possible to send command to the modbus_tk.Simulator thanks to Remote Process Call
  • modbus_tk.modbus.Slave: This class define a modbus slave which is in charge of making the action asked by a modbus query
  • modbus_tk.modbus_tcp.TcpMbap: Defines the information added by the Modbus TCP layer
  • modbus_tk.utils.WorkerThread: A thread which is running an almost-ever loop It can be stopped by calling the stop function
  • object: The most base type
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • modbus_tk.modbus.DuplicatedKeyError: Exception raised when trying to add an object with a key that is already used for another object
        • modbus_tk.modbus.InvalidArgumentError: Exception raised when one argument of a function doesn't meet what is expected
        • modbus_tk.modbus.InvalidModbusBlockError: Exception raised when a modbus block is not valid
        • modbus_tk.modbus.MissingKeyError: Exception raised when trying to get an object with a key that doesn't exist
        • modbus_tk.modbus.ModbusError: Exception raised when the modbus slave returns an error
        • modbus_tk.modbus.ModbusFunctionNotSupportedError: Exception raised when calling a modbus function not supported by modbus_tk
        • modbus_tk.modbus_tcp.ModbusInvalidMbapError: Exception raised when the modbus TCP header doesn't correspond to what is expected
        • modbus_tk.modbus.ModbusInvalidRequestError: Exception raised when the request by the master doesn't fit with the expected format
        • modbus_tk.modbus.ModbusInvalidResponseError: Exception raised when the response sent by the slave doesn't fit with the expected format
        • modbus_tk.modbus.ModbusNotConnectedError: Exception raised when the master is not connected to the slave
        • modbus_tk.modbus.OutOfModbusBlockError: Exception raised when accessing out of a modbus block
        • modbus_tk.modbus.OverlapModbusBlockError: Exception raised when adding modbus block on a memory address range already in use
    • threading._Verbose
      • threading.Thread
        • modbus_tk.simulator.ConsoleInterface: Manage user actions from the console
        • modbus_tk.simulator.RpcInterface: Manage RPC call over TCP/IP thanks to the SocketServer module
